## 1. ペット

### 1.1 データ構造

| フィールド | 型 | 説明 |
|-----------|-----|------|
| id | string/number | 一意識別子 |
| name | string | 表示名（最大6文字・リネーム可・既存データは pet.type でフォールバック） |
| type | enum(10種) | 種類 |
| level | number | ペットレベル |
| hp | number | 体力（上限100） |
| mp | number | スタミナ（上限100） |
| attack | number | 攻撃（上限100） |
| defense | number | 防御（上限100） |
| hunger | number | 満腹度 |
| personality | enum(5種) | 性格（ランダム決定） |
| rarity | string | レア度 |
| imageData | Blob | 元画像（IndexedDB保存） |

### 1.2 ペット種類（10種）

| typeIndex | 種類 | 輪郭特徴 | ゲームへの影響 |
|-----------|------|---------|--------------|
| 0 | ドラゴン系 | 縦横比縦長・エッジ密度中 | なし（図鑑のみ） |
| 1 | 鳥類系 | 重心上・縦横比横長 | なし（図鑑のみ） |
| 2 | 野獣系 | 重心中・エッジ密度高 | なし（図鑑のみ） |
| 3 | スライム系 | 縦横比正方形・エッジ密度低・重心下 | なし（図鑑のみ） |
| 4 | 精霊系 | エッジ密度最低・輪郭不明瞭 | なし（図鑑のみ） |
| 5 | 水棲系 | エッジ中帯・アスペクト中央帯 | なし（図鑑のみ） |
| 6 | 昆虫系 | エッジ高・重心上寄り | なし（図鑑のみ） |
| 7 | 植物系 | 上記いずれにも該当しない残余 | なし（図鑑のみ） |
| 8 | 岩石系 | エッジ極大 | なし（図鑑のみ） |
| 9 | 幻影系 | エッジ極小 | なし（図鑑のみ） |

### 1.3 性格（5種）

| 性格 | 輝度範囲 | 成長補正 |
|------|---------|---------|
| 勇猛 | 最高 | 攻撃成長率↑ |
| 活発 | 高 | MP成長率↑ |
| 強靭 | 中 | HP成長率↑ |
| 堅固 | 低 | 防御成長率↑ |
| 神秘 | 最低 | 全ステ成長率小↑ |

### 1.4 生成ロジック（画像 → ペット）

| 解析ステップ | 手法 | 出力 |
|------------|------|------|
| 色解析 | 平均RGB | 属性（火/水/草/闇/光） |
| 輪郭解析 | エッジ検出 | 種類（10種） |
| 明るさ解析 | 輝度平均 | 性格（5種） |
| ノイズ解析 | 分散 | レア度 |

- 各ステップは独立関数として実装（差し替え可能）
- 同一画像 → 同一Petオブジェクト（再現性あり）
- 実装方式：Canvas API簡易版（ブラウザ完結）

### 1.5 ステータス上昇（餌やり）

- 通常：+1〜+5 ランダム上昇（性格補正を乗算）
- レア成長：低確率で +10
- 確率減衰：上限付近で段階的に減衰

| 現在値（上限比） | 上昇確率倍率 |
|----------------|------------|
| 0〜70% | ×1.0 |
| 71〜90% | ×0.5 |
| 91〜100% | ×0.1 |

- 全ステータス上限は `calcStatCaps()` による可変値（種類別・性格別・レア度別ボーナス加算）。全ステータスが各上限に到達後：HP回復のみ発動（通貨消費あり）

### 1.6 レア度別成長ボーナス確率

| レア度 | レアボーナス発生確率 |
|--------|----------------|
| 伝説 | 30% |
| 英雄 | 20% |
| 希少 | 10% |
| 高級 | 5% |
| 一般 | 2% |

## 4. 訓練（オートバトル）

- 進行：自動連戦（HP0 or 満腹度0 or 中断まで継続）
- 難易度：3段階手動選択
- 中断：ログモーダルの中断ボタンで任意停止。累積結果を表示

| 難易度 | 係数 | 報酬倍率 |
|--------|------|---------|
| やさしい | ×0.5 | ×0.5 |
| ふつう | ×1.0 | ×1.0 |
| むずかしい | ×1.5 | ×1.5 |

### 4.0 連戦ルール

| 項目 | 値 |
|------|-----|
| 連戦継続条件 | HP > 0 かつ 満腹度 > 0 かつ 未中断 |
| 停止理由 | HP0 / 満腹度0 / 中断 |
| 戦闘間の区切り | ログに「── 第N戦 ──」を挿入 |
| 結果表示 | 累積（勝利数/総戦闘数・累積EXP・累積通貨） |

### 4.1 計算式

```
ペット総合力  = HP + MP + 攻撃 + 防御
難易度       = (ユーザーLv × 10 + ペット総合力 × 0.5) × 難易度係数
勝率         = ペット総合力 ÷ 難易度（0.1〜0.9クランプ）
EXP獲得量    = 難易度 × 0.5（勝利時のみ）
通貨報酬     = 8 × ユーザーLv × 報酬倍率（上限200）
```

### 4.2 結果処理

| 結果 | 処理 |
|------|------|
| 勝利 | EXP・通貨獲得・HP減少 |
| 敗北 | HP減少のみ・報酬なし |
| HP0 | 訓練不可・餌やりでHP+20回復後に再開可能 |
| 満腹度0 | 訓練不可・餌やりで回復後に再開可能 |

### 4.3 ターン進行（表示演出）

| 項目 | 値 |
|------|-----|
| ターン数 | 3（固定） |
| 演出種別 | 攻撃ヒット・ミス・被弾・クリティカル・スキル発動 |
| ミス確率 | 20%（演出のみ・勝敗計算に影響しない） |
| クリティカル確率 | 20%（演出のみ・勝敗計算に影響しない） |
| スキル発動時 | ターン1をスキル行に置き換え |
| ターン行待機 | 600ms |
| 結果行待機 | 300ms |

## 14. 見た目変化（進化）

| 項目 | 値 |
|------|-----|
| トリガー | 餌やり後に総合力（HP+MP+ATK+DEF）が閾値を超過したとき |
| stage上限 | 2 |
| 条件判定 | feedPet内でsavePet完了後に判定 |
| imageData | 変更しない（進化前後で同一画像） |
| 演出 | 進化オーバーレイ表示・フレーム枠変化 |

### 進化段階

| stage | 総合力閾値 | メッセージ |
|-------|-----------|-----------|
| 0→1 | 100 | 見た目が鮮やかになった！ |
| 1→2 | 300 | 神々しく輝いている！ |

### 実装制約

- 進化チェックは1回のfeedPetで1段階のみ（閾値を一気に飛び越えても1段階）
- 既存データにevolutionStageフィールドが存在しない場合は `?? 0` で0扱い

### 進化フレーム枠

| stage | クラス | 見た目 |
|-------|--------|--------|
| 0 | なし | 枠なし |
| 1 | `evo-stage-1` | シルバー枠（box-shadow 3px） |
| 2 | `evo-stage-2` | ゴールド枠＋外側グロー |

- 適用箇所：庭・ケージ・訓練（選択欄・ステータス欄・結果オーバーレイ）
- 図鑑には適用しない（種類閲覧用途のため）
- クラス変換は `getEvolutionClass(stage)` ヘルパーで一元管理

## 17. ペット削除（野に放つ）

| 項目 | 値 |
|------|-----|
| 操作箇所 | ケージ画面の「✏️ 編集」ボタンでモード切替・各カード下部に「🌿 野に放つ」ボタンを表示 |
| 確認方式 | overlay-cardスタイルのカスタム確認ダイアログ（ペット名・画像付き） |
| 庭に出ている間 | 削除不可・ボタンをdisabled表示し「庭から外してください」と案内 |
| 削除処理 | `deletePet(id)` でPetストア削除＋User.pets・gardenPetIds同時クリーンアップ |
| 削除後 | 編集モードを終了しrenderCage・renderGardenを呼び出して即時反映 |
| 復元 | 不可（不可逆操作） |
